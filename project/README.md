## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

Если структура проекта отличается от базовой, опишите это здесь.

## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

Примеры (заполните под свой случай):

### 4.1. Модель уже обучена

Артефакты модели (матрица схожести, TF-IDF векторайзер, scaler) сохранены в `artifacts/exp03/` и загружаются сервисом при старте. Переобучение не требуется.

Чтобы воспроизвести эксперименты — запустить ноутбуки в порядке `exp01_eda.ipynb` → `exp02_baseline.ipynb` → `exp03_weight_tuning.ipynb`.

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
# из корня репозитория
cd project
uv run uvicorn src.service.app:app --host 0.0.0.0 --port 8000
```

Или через Docker:

```bash
cd project
docker build -t tour-recommender .
docker run -p 8000:8000 tour-recommender
```

Сервис поднимается на `http://localhost:8000`.

Эндпоинты:
- `GET /health` — статус сервиса
- `GET /tours` — список туров (фильтры: `?city=Istanbul&category=comfort`)
- `POST /predict` — Top-K похожих туров по `tour_id`

Проверка работоспособности:

```bash
curl http://localhost:8000/health
```

Интерактивная документация: `http://localhost:8000/docs`

  - структура кода и конфигураций в целом адекватна;
  - по чеклисту выполнено **не менее 5** пунктов.
